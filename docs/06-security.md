# 六、安全设计

## 6.1 认证安全

### Token 管理

```
┌─────────────────────────────────────────────────────┐
│                Token 安全策略                         │
├─────────────────────────────────────────────────────┤
│                                                     │
│  ✅ Token 只从环境变量读取，不硬编码在配置文件中        │
│     github:                                         │
│       token: "${GITHUB_TOKEN}"                      │
│                                                     │
│  ✅ 配置文件中 Token 字段脱敏显示                      │
│     日志输出中自动替换为 ***                           │
│                                                     │
│  ✅ 支持 GitHub App 认证（比 PAT 更安全）              │
│     · 权限范围可精确控制                              │
│     · Token 自动过期和轮换                            │
│     · 审计日志更完整                                  │
│                                                     │
│  ✅ Token 权限检查                                    │
│     · 启动时检查 Token 权限范围                       │
│     · 操作前预检查所需权限                            │
│     · 权限不足时给出明确提示和修复建议                  │
│                                                     │
│  ❌ 禁止在日志、错误信息、MCP 响应中泄露 Token          │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Token 权限范围要求

| 功能 | 需要的权限 | 最小 scope |
|------|-----------|------------|
| 读仓库/文件 | `contents: read` | `repo` (read) |
| 读 Issue/PR | `issues: read`, `pull_requests: read` | `repo` (read) |
| 创建/更新 Issue | `issues: write` | `repo` |
| 创建/合并 PR | `pull_requests: write` | `repo` |
| 创建 Tag/Release | `contents: write` | `repo` |
| 触发 CI | `actions: write` | `repo` |
| 读 CI 日志 | `actions: read` | `repo` (read) |
| 通知 (API) | `notifications: read` | `notifications` |

## 6.2 操作安全

### 写操作确认机制

```
┌──────────────────────────────────────────────────────┐
│              操作风险分级                               │
├──────────┬──────────┬────────────────────────────────┤
│ 风险等级  │ 操作示例  │ 处理策略                        │
├──────────┼──────────┼────────────────────────────────┤
│ 🟢 低    │ 读文件    │ 直接执行                        │
│          │ 搜代码    │                                │
│          │ 列 Issue  │                                │
├──────────┼──────────┼────────────────────────────────┤
│ 🟡 中    │ 创建 Issue│ 执行 + 日志记录                  │
│          │ 添加评论  │                                │
│          │ 触发 CI   │                                │
├──────────┼──────────┼────────────────────────────────┤
│ 🔴 高    │ 删除 Tag  │ 执行前明确提示                   │
│          │ 合并 PR   │ 日志记录 + 可配置二次确认          │
│          │ 关闭 Issue│                                │
├──────────┼──────────┼────────────────────────────────┤
│ ⚫ 危险  │ 删除仓库  │ 默认禁止                       │
│          │ force push│ 需显式配置 enable_destructive   │
└──────────┴──────────┴────────────────────────────────┘
```

### 配置示例

```yaml
security:
  # 写操作日志审计
  audit_log:
    enabled: true
    path: "./logs/audit.jsonl"
    # 记录所有写操作的完整参数
    log_params: true

  # 危险操作控制
  destructive_operations:
    enabled: false            # 默认禁止危险操作
    allowed_repos: []         # 白名单仓库（即使开启也只允许这些仓库）

  # API 速率限制保护
  rate_limit:
    warn_threshold: 0.8       # 达到 80% 时告警
    pause_threshold: 0.95     # 达到 95% 时暂停写操作
```

## 6.3 传输安全

| 模式 | 安全措施 |
|------|---------|
| **stdio** | 本地进程间通信，默认安全。Token 通过环境变量传递 |
| **SSE** | 支持 TLS（HTTPS），支持 Bearer Token 认证 |
| **Streamable HTTP** | 支持 TLS（HTTPS），支持 Bearer Token 认证 |

## 6.4 数据安全

| 关注点 | 措施 |
|--------|------|
| **缓存数据** | 内存缓存，进程退出即清除；Redis 缓存设置 TTL |
| **配置文件** | Token 字段支持环境变量引用，不落盘明文 |
| **日志脱敏** | 自动检测并替换日志中的 Token、密码等敏感信息 |
| **Webhook 签名** | 接收 Webhook 时验证 `X-Hub-Signature-256` |
