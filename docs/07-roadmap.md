# 七、里程碑与交付规划

## 7.1 分阶段交付策略

```
v0.1 ──→ v0.2 ──→ v0.3 ──→ v0.4 ──→ v1.0
 │        │        │        │        │
```

## 7.4 优先级裁剪方案

如果时间紧张，可按以下顺序裁剪：

| 优先级 | 功能 | 可延后？ |
|--------|------|---------|
| **P0 必须** | M0 基础, M1 读操作, M2 Issue 读写, M5 Release | ❌ |
| **P1 重要** | M3 PR, M4 Code Intelligence, M6 CI/CD | 可延后 |
| **P2 增强** | M7 事件监控, M8 通知系统 | 可延后 |

**最小可用版本（MVP）：** M0 + M1 + M2（读写） + M5，预计 3-4 周可交付。
