# 一、产品概述

## 1.1 产品名称

**GitOps MCP Server**（代号：gitops-mcp）

## 1.2 产品定位

> **一句话定义：** 面向开发者和 DevOps 工程师的 **全功能 GitHub 运维中枢**，通过 MCP 协议让 AI Agent 具备完整的 Git 平台操作能力——从代码阅读、Issue 管理到 CI/CD 流水线触发、Release 发布，实现 **"对话即运维"** 的开发体验。

与 ZRead MCP Server（只读的开源项目知识代理）不同，GitOps MCP Server 是一个 **读写双向、面向运维操作** 的平台级工具。

## 1.3 产品愿景

| 层级 | 描述 |
|------|------|
| **近期愿景** | 让开发者通过自然语言完成 90% 的 GitHub 日常操作，不再需要在浏览器和终端之间切换 |
| **中期愿景** | 成为 GitOps 工作流的 AI 控制面，连接 GitHub、GitLab、Gitea 等多平台，统一管理代码生命周期 |
| **远期愿景** | 构建 AI-native 的 DevOps 自动化生态，实现从需求提出到生产部署的全链路智能化 |

## 1.4 核心价值主张

```
┌─────────────────────────────────────────────────────────────┐
│                    传统工作流                                 │
│                                                             │
│  浏览器打开 GitHub → 找 Issue → 写评论 → 切回终端 →           │
│  改代码 → commit → push → 切浏览器 → 关 Issue →              │
│  打 Tag → 写 Release Notes → 发布 → 回去检查 CI ...          │
│                                                             │
│  😫 上下文切换 7+ 次，耗时 30min+                             │
├─────────────────────────────────────────────────────────────┤
│                    GitOps MCP 工作流                          │
│                                                             │
│  "帮我看一下 #42 这个 issue，修复它，                         │
│   测试通过后打个 v2.1.0 的 tag 并发 release"                  │
│                                                             │
│  😊 一句话搞定，全程在 AI 对话中完成                            │
└─────────────────────────────────────────────────────────────┘
```

## 1.5 竞品对比

| 维度 | ZRead MCP | GitHub CLI (`gh`) | GitOps MCP Server（本产品） |
|------|-----------|-------------------|--------------------------|
| **协议** | MCP | CLI / REST API | MCP |
| **读操作** | ✅ 文档搜索、源码阅读 | ✅ 完整 | ✅ 完整 + 智能索引 |
| **写操作** | ❌ 只读 | ✅ 完整 | ✅ 完整 |
| **Issue 管理** | ❌ | ✅ 基础 | ✅ 高级（批量、模板、自动化） |
| **PR 管理** | ❌ | ✅ 基础 | ✅ 高级（审查、自动合并） |
| **Release 管理** | ❌ | ✅ 基础 | ✅ 高级（智能 Release Notes） |
| **CI/CD 触发** | ❌ | ✅ 基础 | ✅ 高级（监控、重试） |
| **Webhook/监控** | ❌ | ❌ | ✅ 实时监听 |
| **通知系统** | ❌ | ❌ | ✅ 多渠道 |
| **多仓库管理** | ❌ | ⚠️ 需切换 | ✅ 统一视图 |
| **AI 适配** | ✅ MCP 原生 | ❌ 需封装 | ✅ MCP 原生 |
| **部署方式** | 远程托管 | 本地安装 | 本地 / 远程均可 |
| **平台支持** | GitHub | GitHub | GitHub → GitLab / Gitea |

**核心差异化：** 本产品是唯一一个将 **代码阅读 + 运维操作 + 事件监控 + 通知系统** 整合在 MCP 协议下的方案，实现了一站式的 AI-native DevOps 体验。
